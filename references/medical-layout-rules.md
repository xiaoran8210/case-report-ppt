# Medical Layout Rules

These rules override the generic guizang layouts when clinical clarity would otherwise be lost.

## Imaging Sections

- Put images from the same examination session on the same section.
- Keep CT from one timepoint together.
- Keep MRI from one timepoint together.
- Group by both file naming and clinical content, not by file type alone.
- Use the guizang image-grid layouts only when the medical comparison remains obvious at first glance.

## Cross-Timepoint Comparison

- Suggest cross-timepoint comparison during the second revision pass when it would materially improve the case narrative.
- When comparison images share one section, place earlier images above and later images below, or left-to-right if the layout is clearly chronological.
- Label each image group clearly with modality and date.

## Endoscopy And Pathology

- Put GI endoscopy and pathology reports on separate sections unless the user explicitly wants them merged.
- Keep report screenshots and key text conclusions together.
- When the report contains useful mucosal or cellular photo areas, create local magnified views for clarity.
- Make sure the enlarged image area still maps clearly back to the report conclusion.
- If the source is a scanned pathology form, prefer a concise clinical takeaway beside the original image instead of transcribing every line.

## One Question Per Page

- In medical case presentation mode, each section should answer one main question only.
- Do not place general status and disease-onset clue on the same section.
- Do not place initial diagnosis and first MDT question on the same section.
- Do not place surgery narrative and postoperative pathology confirmation on the same section when pathology is clinically important.
- If one section starts explaining a second independent issue, split it.

## Detail Preservation

- Do not over-compress the original case deck just to achieve a shorter total page count.
- For dense diagnostic intake, it is preferable to keep separate sections for endoscopy, pathology, and key imaging if each contributes a different conclusion.
- When the source PPT already uses separate medical logic blocks, preserve that separation unless the merge clearly improves clarity.
- Use image-backed sections for endoscopy and imaging whenever source images are available.

## Core Discussion Placement

- Move the core case question, key MDT reflection, or teaching-point synthesis toward the end of the deck.
- Prefer placing that summary on the penultimate section, after the factual disease course, pathology confirmation, and timeline have already been presented.

## Text Density

- Guizang layouts are editorial, not training-course dense. Split overloaded medical content across more sections instead of stuffing one page.
- Prefer short clinical conclusions, bold callouts, and clearly separated evidence blocks.
- If a medical page needs many details, use a dark/light alternation and multi-section rhythm rather than shrinking type to fit.

## Theme Constraint

- Even though guizang supports strong editorial themes, do not let visual style overpower pathology, imaging, or staging information.
- For medical case reports, prefer the more restrained preset themes unless the user explicitly wants a more expressive deck.

## Cover Image

- The cover image may be generated, but it must remain medically relevant and restrained.
- Favor anatomy, imaging texture, pathology-inspired abstraction, or subtle clinical atmosphere.
- Avoid sensational rendering or decorative imagery that competes with the title.

## Timeline

- The timeline is required.
- Default to a horizontal milestone structure.
- Upgrade to a multi-track structure when the case contains multiple reassessments, recurrence phases, or parallel pathology/imaging/treatment threads.
- The timeline must explicitly surface any diagnostic ambiguity that matters to the teaching point, such as possible synchronous versus metachronous disease.
